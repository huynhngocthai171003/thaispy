Phân chia mã trong Python 15 phút đọc

Trong Python, việc tổ chức và phân chia mã nguồn giúp chương trình dễ quản lý, bảo trì, mở rộng và tái sử dụng. Python cung cấp nhiều cách để phân chia mã: module, package, library và framework.

1. Khái niệm cơ bản

  • Module: Một file .py chứa hàm, biến, lớp có thể tái sử dụng.
  • Package: Một thư mục chứa nhiều module, có file __init__.py.
  • Library: Tập hợp các module/package hỗ trợ một chức năng cụ thể.
  • Framework: Bộ khung phần mềm toàn diện giúp xây dựng ứng dụng nhanh và chuẩn hóa hơn.

2. Module – Định nghĩa, Ví dụ và Giải thích

Module là cách phân chia mã đơn giản nhất – một tệp Python chứa các hàm, biến hoặc lớp có thể tái sử dụng.

# mymath.py
def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

# main.py
import mymath

print(mymath.add(3, 4))        # Output: 7
print(mymath.subtract(10, 2))  # Output: 8

Giải thích: Bạn có thể tái sử dụng các hàm từ mymath.py bằng cách import nó vào file khác.

3. Sử dụng từ khóa as để đổi tên module

import mymath as mm

print(mm.add(5, 6))  # Output: 11

Giúp tên module ngắn gọn hơn, đặc biệt khi module có tên dài.

4. Built-in Modules – Các module có sẵn trong Python

Python cung cấp rất nhiều module tích hợp sẵn (built-in) mà bạn có thể sử dụng trực tiếp mà không cần cài đặt thêm bằng pip. Những module này giúp bạn xử lý nhiều tác vụ phổ biến như tính toán, làm việc với ngày giờ, ngẫu nhiên, và tương tác với hệ thống.

Dưới đây là một số ví dụ về các built-in module thường dùng:

import math, random, datetime, sys

# Sử dụng module math để tính toán
print(math.sqrt(25))              # 5.0 - căn bậc hai của 25

# Sử dụng module random để chọn ngẫu nhiên
print(random.choice([1, 2, 3]))   # Trả về một phần tử ngẫu nhiên từ danh sách

# Sử dụng module datetime để lấy thời gian hiện tại
print(datetime.datetime.now())    # Ví dụ: 2025-08-03 09:25:00

# Sử dụng module sys để lấy thông tin phiên bản Python
print(sys.version)                # Ví dụ: 3.10.12 (main, Jun 5 2025)

Giải thích chi tiết:

math: Cung cấp các hàm toán học như căn bậc hai (sqrt), lũy thừa, lượng giác, v.v.

random: Tạo ra số ngẫu nhiên hoặc chọn ngẫu nhiên phần tử từ danh sách.

datetime: Làm việc với ngày và giờ, hỗ trợ xử lý thời gian hiện tại, ngày tháng, định dạng thời gian.

sys: Giao tiếp với hệ thống – ví dụ như truy cập thông tin phiên bản Python, đường dẫn module, và các tham số dòng lệnh.

Những module này được cài đặt sẵn khi bạn cài Python, vì vậy bạn chỉ cần import để sử dụng mà không phải cài thêm gì cả.

5. Vị trí và Thứ tự Ưu tiên khi tìm Module

Khi bạn sử dụng lệnh import ten_module, Python sẽ tìm module đó theo một thứ tự được xác định trong biến sys.path. Đây là danh sách các thư mục mà Python sẽ kiểm tra khi cần nạp một module.

Cụ thể, thứ tự tìm kiếm như sau:

Thư mục hiện tại (Current Working Directory):
Đây là thư mục chứa file Python bạn đang chạy. Nếu trong thư mục này có file mymodule.py, Python sẽ ưu tiên import file đó trước.

Ví dụ:

/home/user/project/
├── main.py
└── mymodule.py
# main.py
import mymodule  # Tìm thấy mymodule.py trong thư mục hiện tại

Các thư viện đã cài bằng pip (Site-packages):
Nếu không tìm thấy module trong thư mục hiện tại, Python tiếp tục tìm trong thư mục chứa các thư viện bạn đã cài bằng pip.

Đường dẫn thường giống như:

/usr/local/lib/python3.x/site-packages/

Ví dụ:

import requests  # Python tìm thấy trong thư mục site-packages

Thư viện chuẩn (Standard Library):
Nếu vẫn chưa tìm thấy, Python tiếp tục kiểm tra các module tích hợp sẵn như math, datetime, sys, v.v.

Ví dụ:

import math  # Python tìm thấy trong thư viện chuẩn

Kiểm tra toàn bộ đường dẫn tìm kiếm:

import sys
for p in sys.path:
    print(p)

Lệnh trên sẽ in ra danh sách thư mục theo đúng thứ tự mà Python sẽ kiểm tra khi tìm module.

Lưu ý quan trọng:

Giả sử bạn vô tình đặt tên một file trùng với module chuẩn của Python, ví dụ random.py:

# random.py
import random
print(random.randint(1, 10))

Python sẽ cố gắng import chính file random.py của bạn, không phải module chuẩn. Điều này dẫn đến lỗi:

AttributeError: module 'random' has no attribute 'randint'

Thêm thư mục tùy chỉnh vào đường dẫn tìm kiếm:

import sys
sys.path.append('/duong_dan/module_cua_ban')

Cách này hữu ích khi bạn muốn dùng module nằm ở nơi khác, không nằm trong các thư mục mặc định.

Tóm tắt: Thứ tự tìm module là:

1. Thư mục hiện tại

2. Thư viện đã cài bằng pip

3. Thư viện chuẩn (built-in)

Bạn có thể hiểu đơn giản: Python sẽ tìm trong "nhà mình" trước, rồi đến "kho hàng đã mua thêm", sau đó mới đến "hàng chuẩn nhà sản xuất".

6. Package – Định nghĩa, Ví dụ và Giải thích

Package là thư mục chứa nhiều module, có file __init__.py để Python nhận diện là package.


calculator/
├── __init__.py
├── basic.py
└── advanced.py

# calculator/basic.py
def add(a, b):
    return a + b

# main.py
from calculator import basic
print(basic.add(2, 3))  # Output: 5

Giải thích: Package giúp nhóm các module có chức năng liên quan lại với nhau.

7. External Module – Thư viện bên ngoài và Cộng đồng Python

Ngoài các module sẵn có, Python có cộng đồng lớn chia sẻ hàng trăm nghìn thư viện mã nguồn mở. Cài đặt bằng pip:

pip install requests
import requests

res = requests.get("https://httpbin.org/get")
print(res.status_code)

Giải thích: requests là một thư viện giúp gửi HTTP request dễ dàng.

8. Library vs Framework – Sự khác biệt

Library (thư viện) và Framework (khung phần mềm) đều là các thành phần hỗ trợ lập trình viên, nhưng có cách hoạt động và mức độ kiểm soát hoàn toàn khác nhau.

  • Library: Bạn chủ động gọi các hàm, lớp, hoặc đối tượng trong thư viện để sử dụng. Luồng điều khiển do bạn quyết định. Ví dụ: gọi hàm từ requests, pandas, numpy khi cần.
  • Framework: Framework đưa ra cấu trúc và gọi ngược lại mã của bạn theo quy tắc định sẵn. Bạn phải viết mã phù hợp với luồng và cấu trúc mà framework cung cấp. Đây là nguyên lý gọi là Inversion of Control (đảo ngược quyền điều khiển).

Ví dụ:

Library – Bạn gọi hàm thư viện khi cần:

import requests

response = requests.get('https://httpbin.org/get')
print(response.status_code)

Trong ví dụ này, bạn chủ động gọi requests.get(). Bạn điều khiển quá trình xử lý.

Framework – Framework gọi hàm bạn khi phù hợp:

from flask import Flask

app = Flask(__name__)

@app.route('/')
def home():
    return "Hello, world!"

app.run()

Ở đây, bạn không tự gọi hàm home(). Flask tự động gọi nó khi có yêu cầu HTTP đến /.

So sánh nhanh:
Tiêu chí Library Framework
Điều khiển chương trình Bạn điều khiển Framework điều khiển
Luồng thực thi Do lập trình viên quyết định Định nghĩa sẵn bởi framework
Cấu trúc bắt buộc Không Có
Ví dụ NumPy, Pandas, Requests Django, Flask, FastAPI

Tóm tắt dễ nhớ: "Thư viện là công cụ bạn gọi. Framework là công cụ gọi bạn!"

10. Tổng kết

  • Module: Tệp .py chứa mã tái sử dụng.
  • Package: Thư mục gồm nhiều module.
  • Library: Nhóm module theo chức năng chuyên biệt.
  • Framework: Hệ thống xây dựng ứng dụng hoàn chỉnh.
  • Dùng import, from, as để nạp module.
  • Có thể sử dụng pip để mở rộng với hàng ngàn thư viện ngoài.

Hiểu rõ cách phân chia mã trong Python sẽ giúp bạn xây dựng các chương trình có tổ chức, dễ mở rộng và dễ bảo trì.
